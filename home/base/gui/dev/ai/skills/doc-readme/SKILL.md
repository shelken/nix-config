# doc-readme

## 作用

限定如何写README, 如果项目已经有了, 审查README是否符合以下规范, 先列出建议变更项, 标上序号, 再询问用户是否做出变更

## 规则

- README 应该简洁明了，易于理解
- 默认放在项目根目录下
- 有`功能`块, 列出最核心功能和简单brief, 禁止长篇大论和大量小功能点
- 有`快速上手`块, 引导用户快速上手
- 有展示核心配置示例, 和配置的表格(描述默认值和作用)
- 如果项目有`docs/user-guide/`, 链接并在文档指引
- 有`贡献`块, 引导用户如何贡献
- 有`许可证`块, 说明项目的许可证
