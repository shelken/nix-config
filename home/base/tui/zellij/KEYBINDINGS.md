# Zellij 快捷键参考

> 配置文件位置：`~/.config/zellij/config.kdl`

## 模式切换

| 快捷键     | 模式    | 说明                       |
| ---------- | ------- | -------------------------- |
| `Ctrl + g` | Locked  | 锁定模式（屏蔽所有快捷键） |
| `Ctrl + p` | Pane    | Pane 操作模式              |
| `Ctrl + n` | Resize  | 调整大小模式               |
| `Ctrl + h` | Move    | 移动/交换 Pane 模式 ⚠️     |
| `Ctrl + t` | Tab     | Tab 操作模式               |
| `Ctrl + s` | Scroll  | 滚动模式                   |
| `Ctrl + o` | Session | 会话模式                   |
| `Ctrl + b` | Tmux    | Tmux 兼容模式              |

> ⚠️ **注意**：`Ctrl + h` 当前在 `keybinds.kdl` 中被
> `unbind`，用于避免与 vim 冲突。如需使用，请移除该 unbind。

## 布局切换

| 快捷键    | 功能       | 说明             |
| --------- | ---------- | ---------------- |
| `Alt + [` | 上一个布局 | 切换 swap layout |
| `Alt + ]` | 下一个布局 | 切换 swap layout |

### 可用布局（在 `basic.swap.kdl` 中定义）

1. **默认布局** - 平铺布局
2. **3up** - 3 个垂直排列的 pane
3. **vertical** - 垂直分割（主 pane 在左，其他在右堆叠）
4. **horizontal** - 水平排列
5. **stacked** - 堆叠布局（左侧一个，右侧 pane 堆叠显示）

> 💡 **示例**：当前是上下两个 pane 时，按 `Alt + ]` 可切换到左右布局。

## 常见问题

### Q: `Ctrl + h` 进入 Move 模式没反应？

**A**: 在 `keybinds.kdl` 第 8 行，`Ctrl + h` 被 unbind 了（为了 vim 兼容）。

**解决方案**：

1. 如果不用 vim 的 `Ctrl + h`，编辑 `keybinds.kdl`：

   ```kdl
   unbind "Ctrl q" "Ctrl s" "Ctrl o"  // 移除 "Ctrl h"
   ```

2. 或者使用其他快捷键，在 `config.kdl` 中添加：
   ```kdl
   shared_except "move" "locked" {
       bind "Ctrl m" { SwitchToMode "Move"; }  // 或其他不冲突的键
   }
   ```

---

### Q: 如何快速切换上下布局为左右布局？

**A**: 按 `Alt + ]` 或 `Alt + [` 切换布局，循环遍历可用布局。

---

### Q: Resize 调整步长太大，如何微调？

**A**: Zellij 的 resize 步长是固定的。如需微调，可快速多次按 `h/j/k/l` 或 `Shift + h/j/k/l`。
